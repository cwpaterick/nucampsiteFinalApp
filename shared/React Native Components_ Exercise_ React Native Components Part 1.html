    3. Exercise: React Native Components Part 1

    READ THIS FIRST:

    In the latest versions of the Expo CLI, the Expo team has very recently deprecated a feature you'll see in the video below called the Expo DevTools, a web browser interface running your app locally for development. Thus, contrary to what you see in the video below, you will not see the Expo DevTools browser page when you use the command npm run android or npm start in this exercise. 

    This is expected, and the video will be updated soon to reflect this. Instead of using the Expo DevTools browser page to connect to your AVD, you will use the bash terminal instead. After you enter npm run android or npm start, you should be shown a list of options. If you ran npm run android and you have an AVD already open your app should automatically open in the AVD. You can press a to connect to your AVD again if you need to restart the app. Or, if you are using a mobile device and ran npm start, use the QR code shown in the bash terminal to connect.

    Additionally, we recently updated this course to provide a package file for you to install, rather than individual packages.  The video will be updated soon, but until then, DO NOT INSTALL ANY PACKAGES, as this has already been done for you.

    Please refer to the written instructions below have been updated, so please follow these instructions instead of the video.


    Click here if you wish to open this video in its own window.



    What You'll Do

    • Use UI components from both React Native's set of built-in components and RNE to begin building the UI for your app.



    Instructions

    Exercise Resources

    images.zip
campsites.js


    Set up images and campsites data

    • First, open VS Code to the nucampsite/ folder that is inside the 4-React-Native/ folder. Make sure you are opening the nucampsite/ folder and NOT the 4-React-Native/ folder!
    • Next, create two new folders in the nucampsite/ folder: one named screens, and one named shared.
    • Download the images.zip file given above, unzip it, and move the images/ folder into the assets/ folder. Alternatively, you can copy over the images/ folder you used in the React course, from either the React project's public/assets/ folder or the json-server's public/ folder, into the assets/ folder. It is the same set of files.
    • Next, download the campsites.js file given above and move it into the shared/ folder. This is different than the campsites.js file inside the React project's shared/ folder, so make sure you use the one provided in this exercise.

    *** Note: You do NOT need to install React Native Elements as directed in the video, because it has already been installed from the package file. The video will be updated soon to reflect this, but for now, you can continue to the next step.***


    Create MainComponent.js

    • Add a new file named MainComponent.js in the screens/ folder.
    • Import the useState hook from 'react'.
    • Import the CAMPSITES array from the shared/ folder.
    • Import the DirectoryScreen component from './DirectoryScreen'. We will create the DirectoryScreen component next.
    • Create a constant named Main set equal to an arrow function and export Main at the bottom of this file outside of the arrow function body.
    • Create a state variable using array destructuring syntax with the first value of campsites, and the second value of setCampsites set equal to the useState hook passing in the CAMPSITES array as an argument to useState.

    const [campsites, setCampsites] = useState(CAMPSITES);
    • Return the DirectoryScreen tag from the return statement passing a single prop named campsites equal to the campsites state variable.

    return <DirectoryScreen campsites={campsites} />;


    Create DirectoryScreen.js

    • Then, add another file named DirectoryScreen.js in the screens/ folder.
    • Import FlatList from 'react-native'.
    • Import Avatar and ListItem from 'react-native-elements'.
    import { FlatList } from 'react-native';
import { Avatar, ListItem } from 'react-native-elements';
    • Create a constant named DirectoryScreen equal to an arrow function, passing in one parameter named props.
    • Export the DirectoryScreen variable at the bottom of the file outside the arrow function body.
    • Return the FlatList component from the return statement, passing the three props, data, renderItem and keyExtractor.
    • Set the data prop equal to the campsite prop passed in by the Main component like this props.campsite.
    • Set the renderItem prop equal to a variable named renderDirectoryItem and create an arrow function above the return statement but still inside the DirectoryScreen arrow function body. Set this arrow function equal to a constant with the same name of renderDirectoryItem.
    • Destructure a property named item in the parameter list of the renderDirectoryItem arrow function, giving it a local name of campsite like this:
    const renderDirectoryItem = ({ item: campsite }) => {
    • Inside the renderDirectoryItem arrow function body set up a return statement and return a ListItem component using an opening and closing tag.
    • Inside the opening and closing tag of the ListItem create a self-closing tag with the Avatar component passing it two props source and rounded setting source equal to campsite.image and leaving rounded equal to nothing. (We can set the rounded prop equal to nothing leaving out the equal sign because it is a boolean value which is shorthand for setting it equal to true).
    • After the Avatar component but still between the opening and closing tags of ListItem add a component named ListItem.Content using an opening and closing tag.
    • Between the ListItem.Content opening and closing tag create an opening and closing tag of ListItem.Title passing campsite.name between the opening and closing tags in curly brackets.
    • Still inside the ListItem.Content opening and closing tag create another opening and closing tag of ListItem.Subtitle, passing campsite.description between the opening and closing tags in curly brackets.

    return (
    <ListItem>
        <Avatar source={campsite.image} rounded />
        <ListItem.Content>
            <ListItem.Title>{campsite.name}</ListItem.Title>
            <ListItem.Subtitle>
                {campsite.description}
            </ListItem.Subtitle>
        </ListItem.Content>
    </ListItem>
);
    • Set the keyExtractor prop to an inline arrow function with a parameter item that returns item.id.toString() like this:
    keyExtractor={(item) => item.id.toString()}


    Update App.js

    • Next, update App.js as follows:

    import Main from './screens/MainComponent';

export default function App() {
    return <Main />;
}
    • Here we made the following changes:
      • Deleted old imports and replaced with a single import for Main from './screens/MainComponent'.
      • Deleted the style sheet at the bottom of the file.
      • Replaced everything in the return statement with a single tag of <Main/>.


    Test and commit

    • Run expo start in your bash terminal from inside your 4-React-Native/nucampsite/ folder, if it is not already running, load your Android emulator, and connect to the emulator. Confirm that the application shows the Directory, which lists all the campsites in the campsites.js file. 
    • OPTIONAL: Do a Git commit with the message "Components Part 1".
